# Checkpoints

- Every promised behavior from docs/acceptance_contract.json is represented in specs, implementation, tests, and evidence.
- Each feature has executable tests or scenario evidence.
- Every user-visible control, command, route, form, API endpoint, and background job is wired to real behavior.
- verification/traceability.md maps entry points and acceptance criteria to implementation paths and tests.
- `bash verify.sh` exits 0 before and after marking features done.
- Functional review reaches the scenario threshold.
- The final report lists wiring evidence and known gaps. Known gaps mean not done.
